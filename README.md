# UniHR

Консольное приложение (C# .NET) для управления кадровыми данными. Проект разработан для демонстрации принципов объектно-ориентированного программирования (ООП) и моделирования корпоративных бизнес-процессов.

## 🚀 Запуск проекта

- **Запуск приложения:** `dotnet run` (находясь в корневой папке проекта)
- **Запуск Unit-тестов:** `dotnet test` (находясь в корневой папке проекта)

## 📊 Аналитический обзор архитектуры

* **Атрибуты:** В объектах класса `Worker` хранятся данные о сотруднике: `FullName` (наследуется от базового класса), `Position` (должность), `Department` (факультет/отдел), `HireDate` (дата приема) и `Salary` s(оклад).
* **Методы:** * `GetExperience()`: расчет точного стажа (в годах, месяцах и днях).
  * `UpdatePositionAndSalary()`: изменение кадровых данных.
  * `EstimateSalaryFor2026()`: авто-расчет актуальной рыночной зарплаты Синергии.
  * `RequestSoftwareBuy()` и `CollaborateWith()`: симуляция корпоративных процессов.
* **Абстракция:** Создан базовый абстрактный класс `Person`. Он выделяет общую концепцию "человека" в системе, скрывая детали конкретной реализации.
* **Инкапсуляция:** Атрибут зарплаты жестко защищен. Данные хранятся в приватном поле `_salary`, а доступ извне осуществляется через публичное свойство `Salary` со встроенной логикой валидации: если попытаться установить зарплату меньше нуля, система автоматически установит МРОТ (27 093 руб.).
* **Наследование:** Класс `Worker` наследуется от базового класса `Person` (`public class Worker : Person`). Это избавляет от дублирования кода: работник автоматически получает поле `FullName` от родителя, расширяя его своими профессиональными характеристиками.
* **Полиморфизм:** * *Переопределение:* Метод `DisplayInfo()` объявлен как абстрактный в `Person` и переопределен (`override`) в `Worker` под конкретные нужды вывода.
  * *Перегрузка:* Класс имеет несколько версий конструктора (по умолчанию, частичный, полный), которые принимают разное количество аргументов.
* **Соответствие требованиям:** Код полностью закрывает требования Кейс-задачи 3. Реализован ввод с клавиатуры, поиск по стажу (в годах и месяцах), вывод ошибки при отсутствии сотрудников, использование коллекции `List<Worker>`, наличие деструктора `~Worker()` и разделение на пространства имен (модули).
* **Тестирование:** * В приложении реализованы независимые юнит-тесты (xUnit).
  * Валидация `string.IsNullOrWhiteSpace()` блокирует создание сотрудников без имени и должности.
  * `DateTime.TryParseExact()` предотвращает сбой программы при вводе некорректных дат.
  * Проверка `universityStaff.Count >= 2` защищает от ошибки доступа к памяти (*Out of Bounds*) при вызове демо-методов.
* **Документация:** Код соответствует стандартам *Clean Code*: переменные и методы названы семантически понятно, логика разбита на блоки с XML-комментариями.
* **Итерация:** В процессе разработки базовая структура была пересмотрена. Вместо полностью ручного ввода данных был внедрен метод `EstimateSalaryFor2026()` для автоматической оценки зарплаты, а также добавлены методы для последующего анализа корпоративной среды (проблем бюрократии и коммуникации).
